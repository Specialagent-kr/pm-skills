## A/B 테스트 분석 (A/B Test Analysis)

통계적 엄격함을 바탕으로 A/B 테스트 결과를 평가하고, 발견된 사항을 명확한 제품 의사결정으로 전환합니다.

### 맥락

당신은 **$ARGUMENTS**를 위한 A/B 테스트 결과를 분석하고 있습니다.

사용자가 데이터 파일(CSV, Excel 또는 분석 도구 내보내기 등)을 제공하면 이를 직접 읽고 분석하십시오. 필요한 경우 통계 계산을 위한 파이썬 스크립트를 생성하십시오.

### 지침

1. **실험 이해**:
   - 가설은 무엇이었는가?
   - 무엇이 변경되었는가 (변체)?
   - 1차 지표(Primary metric)는 무엇인가? 가드레일 지표(Guardrail metrics)가 있는가?
   - 테스트가 얼마나 지속되었는가?
   - 트래픽 할당 비율은 어떠한가?

2. **테스트 설정 검증**:
   - **샘플 크기**: 기대하는 효과 크기 대비 샘플이 충분히 큰가?
     - 공식: n = (Z²α/2 × 2 × p × (1-p)) / MDE²
     - 검정력(Power)이 낮은 경우(<80%) 표시하십시오.
   - **지속 기간**: 테스트가 최소 1-2회의 전체 비즈니스 사이클 동안 실행되었는가?
   - **무작위 배정**: 샘플 비율 불일치(Sample Ratio Mismatch, SRM)의 징후가 있는가?
   - **신규성/적응 효과(Novelty/primacy effects)**: 초기 행동 변화가 안정화될 만큼 충분한 시간이 지났는가?

3. **통계적 유의성 계산**:
   - 대조군(Control)과 변체(Variant)의 **전환율**
   - **상대적 리프트(Relative lift)**: (변체 - 대조군) / 대조군 × 100
   - **p-값(p-value)**: 양측 Z-검정 또는 카이제곱 검정 사용
   - **신뢰 구간(Confidence interval)**: 차이에 대한 95% 신뢰 구간
   - **통계적 유의성**: p < 0.05 인가?
   - **실무적 유의성**: 리프트가 비즈니스 관점에서 의미가 있는가?

   사용자가 원천 데이터를 제공하는 경우, 이를 계산하기 위한 파이썬 스크립트를 생성하고 실행하십시오.

4. **가드레일 지표 확인**:
   - 가드레일 지표(매출, 인게이지먼트, 페이지 로드 시간 등)가 악화되었는가?
   - 1차 지표가 승리했더라도 가드레일 지표가 악화되었다면 진정한 승리로 보기 어려울 수 있습니다.

5. **결과 해석**:

   | 결과 | 권장 사항 |
   |---|---|
   | 유의미한 긍정적 리프트, 가드레일 문제 없음 | **배포 (Ship it)** — 100% 출시 |
   | 유의미한 긍정적 리프트, 가드레일 우려 있음 | **조사 (Investigate)** — 배포 전 트래이드오프 파악 |
   | 유의미하지 않으나 긍정적 추세 | **테스트 연장** — 더 많은 데이터나 더 큰 효과 필요 |
   | 유의미하지 않고 평이함 | **테스트 중단** — 의미 있는 차이 발견되지 않음 |
   | 유의미한 부정적 리프트 | **배포 안 함** — 대조군으로 복구 후 원인 분석 |

6. **분석 요약 제공**:

   ```
   ## A/B 테스트 결과: [테스트 이름]

   **가설**: [기대했던 사항]
   **지속 기간**: [X일] | **샘플**: [대조군 N명 / 변체 M명]

   | 지표 | 대조군 | 변체 | 리프트 | p-값 | 유의미한가? |
   |---|---|---|---|---|---|
   | [1차 지표] | X% | Y% | +Z% | 0.0X | 예/아니오 |
   | [가드레일] | ... | ... | ... | ... | ... |

   **권장 사항**: [배포 / 연장 / 중단 / 조사]
   **근거**: [이유]
   **이후 단계**: [수행할 작업]
   ```

단계별로 생각하십시오. 마크다운으로 저장하십시오. 원천 데이터가 제공되면 계산을 위한 파이썬 스크립트를 생성하십시오.
