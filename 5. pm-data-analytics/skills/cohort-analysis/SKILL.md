# 코호트 분석 및 유지율 탐색기 (Cohort Analysis & Retention Explorer)

## 목적

사용자 행동, 기능 도입 및 장기적인 참여 트렌드를 파악하기 위해 코호트별로 사용자 참여 및 유지 패턴을 분석합니다. 정량적 통찰력과 정성적 연구 권장 사항을 결합합니다.

## 작동 방식

### 1단계: 데이터 읽기 및 검증

- 사용자 코호트 정보가 포함된 CSV, Excel 또는 JSON 데이터 파일을 수락합니다.
- 데이터 구조를 확인합니다: 코호트 식별자, 시기(Period), 참여 지표 등.
- 누락된 값 및 데이터 품질 문제를 확인합니다.
- 주요 통계 요약(코호트 크기, 날짜 범위, 사용 가능한 지표)을 제공합니다.

### 2단계: 정량적 분석 생성

- 코호트 유지율(Retention rates) 및 참여 패턴을 계산합니다.
- 유지 곡선(Retention curves), 이탈 패턴 및 이상 징후를 식별합니다.
- 코호트별 기능 도입률(Feature adoption rates)을 계산합니다.
- 전월 대비(MoM) 또는 기간별 변화를 계산합니다.
- 요청 시 pandas 및 numpy를 사용한 파이썬 분석 스크립트를 생성합니다.

### 3단계: 시각화 생성

- 유지율 히트맵(코호트 vs 시기)을 생성합니다.
- 코호트 진행률을 보여주는 선 그래프를 만듭니다.
- 기능 도입 비교 차트를 구축합니다.
- 이탈 지점 및 참여 트렌드를 시각화합니다.
- 대화형 차트 또는 정적 이미지로 출력합니다.

### 4단계: 통찰 및 패턴 식별

- 하나 이상의 중요한 패턴을 포착합니다:
  - 특정 코호트에서의 초기 이탈(Churn)
  - 후기 단계의 참여 변화
  - 기능 도입 클러스터
  - 계절성 또는 시간적 트렌드
- 놀라운 발견 사항이나 편차를 강조합니다.
- 기준선(Baseline)을 설정하기 위해 코호트 성과를 비교합니다.

### 5단계: 후속 연구 제안

- 정성적 연구 방법을 권장합니다:
  - 이탈 사용자 대상의 타겟 인터뷰
  - 참여도가 높은 코호트 대상의 기능 사용 설문 조사
  - 주요 상호작용 패턴의 세션 리플레이(Session replays) 분석
  - 고유지율 vs 저유지율 코호트 분석(Win/Loss analysis)
- 후속 정량 연구를 설계합니다.
- A/B 테스트 또는 기능 실험을 제안합니다.

## 사용 예시

**예시 1: CSV 데이터 업로드**

```
cohort_month, weeks_active, user_id, feature_x_usage, engagement_score 컬럼이 포함된 
cohort_engagement.csv 업로드

요청: "유지율 패턴을 분석하고 2025년 4분기 코호트가 3분기에 비해 실적이 저조한 이유를 파악해줘"
```

**예시 2: 데이터 형식 설명**

```
"2025년 1월부터 12월까지의 월별 사용자 코호트 데이터가 있습니다. 
각 행에는 코호트 날짜, 사용자 ID, 구매 빈도, 지원 티켓 수가 포함되어 있습니다. 
어떤 코호트가 가장 좋은 장기 유지율을 보이는지 분석해줘."
```

**예시 3: 기능 도입 분석**

```
코호트 도입 데이터가 포함된 feature_usage.xlsx 업로드

요청: "새 기능에 대한 코호트별 도입 곡선을 비교해줘. 
어떤 코호트가 가장 빨리 도입했는가? 어떤 패턴이 있는가?"
```

## 주요 기능

- **데이터 읽기**: CSV, Excel, JSON, SQL 쿼리 결과 가져오기
- **유지율 분석**: 시간에 따른 유지율 계산 및 시각화
- **코호트 비교**: 코호트 그룹 간 지표 비교
- **이상 징후 탐지**: 비정상적인 패턴이나 이탈 지점 표시
- **파이썬 스크립트**: 지속적인 분석을 위한 재사용 가능한 코드 생성
- **시각화**: 히트맵, 차트 및 대화형 대시보드 생성
- **연구 설계**: 타겟 후속 연구 및 인터뷰 접근 방식 제안
- **통계 요약**: 정량적 지표 및 상관관계 분석 제공

## 최상의 결과를 위한 팁

1. **시간 차원 포함**: 여러 시기에 걸친 데이터를 제공하십시오.
2. **코호트 명확성**: 코호트 그룹화 기준(가입 월, 기능 출시일 등)을 명확히 하십시오.
3. **맥락 제공**: 해당 기간 동안의 제품 변경, 출시 또는 이벤트를 설명하십시오.
4. **다양한 지표**: 유지율, 참여도, 기능 사용량, 매출 등을 포함하십시오.
5. **충분한 데이터**: 의미 있는 패턴 식별을 위해 최소 3-4개의 코호트가 필요합니다.
6. **특정 출력 요청**: 시각화, 파이썬 스크립트 또는 연구 권장 사항 중 필요한 것을 요청하십시오.

## 출력 형식

다음을 받게 됩니다:

- **데이터 요약**: 코호트 개요 및 데이터 품질 평가
- **정량적 분석 결과**: 주요 지표, 유지율 및 트렌드 분석
- **시각화**: 유지 곡선, 도입 패턴을 보여주는 차트
- **패턴 식별**: 데이터에서 발견된 2-3가지 주요 통찰
- **연구 권장 사항**: 구체적인 정성적 및 정량적 후속 조치
- **분석 스크립트** (요청 시): 재현 가능한 분석을 위한 파이썬 코드
- **이후 단계**: 발견된 사항에 기반한 우선순위 지정된 실행 과제
