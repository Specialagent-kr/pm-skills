# SQL 쿼리 생성기 (SQL Query Generator)

## 목적

자연어 요구 사항을 여러 데이터베이스 플랫폼에 최적화된 SQL 쿼리로 변환합니다. 이 스킬은 제품 관리자, 분석가 및 엔지니어가 수동 구문 작업 없이 정확한 쿼리를 생성하도록 돕습니다.

## 작동 방식

### 1단계: 데이터베이스 스키마 이해

- 스키마 파일(SQL, 문서 또는 다이어그램 설명)을 제공하면 이를 읽고 분석합니다.
- 테이블 이름, 컬럼 정의, 데이터 타입 및 관계를 추출합니다.
- 기본 키(Primary key), 외래 키(Foreign key) 및 인덱싱 전략을 식별합니다.

### 2단계: 요청 처리

- 추출하거나 분석해야 하는 정확한 데이터를 명확히 합니다.
- SQL 방언(BigQuery, PostgreSQL, MySQL, Snowflake 등)을 확인합니다.
- 추가 요구 사항(필터, 집계, 정렬 등)을 확인합니다.

### 3단계: 최적화된 쿼리 생성

- 데이터베이스 구조를 활용하는 효율적인 SQL을 작성합니다.
- 복잡한 로직을 설명하는 주석을 포함합니다.
- 대규모 데이터셋에 대한 성능 고려 사항을 추가합니다.
- 해당되는 경우 대안적인 접근 방식을 제공합니다.

### 4단계: 설명 및 테스트

- 쿼리 로직을 평이한 언어로 설명합니다.
- 결과 테스트 또는 검증 방법을 제안합니다.
- 성능 최적화를 위한 팁을 제공합니다.
- 원하는 경우 테스트 스크립트나 샘플 데이터를 생성합니다.

## 사용 예시

**예시 1: 스키마 파일로부터 쿼리 생성**

```
database_schema.sql 파일을 업로드하고 다음과 같이 요청:
"지난 30일 동안 가입하고 최소 5번의 활성 세션을 가진 사용자를 찾는 쿼리 생성해줘"
```

**예시 2: 다이어그램 설명으로부터 쿼리 생성**

```
"직접 설명하는 데이터베이스 구조: Users 테이블 (id, email, created_at), Sessions 테이블
(id, user_id, timestamp, duration). 2026년 1월의 사용자별 평균 세션 지속 시간을 구하는 쿼리를 생성해줘."
```

**예시 3: 복잡한 분석 쿼리**

```
"지역 및 고객 등급별 매출을 분석하고 전년 대비 성장률을 포함하는 BigQuery 쿼리를 작성해줘."
```

## 주요 기능

- **다양한 방언 지원**: BigQuery, PostgreSQL, MySQL, Snowflake, SQL Server 지원
- **파일 읽기**: 스키마 파일, SQL 덤프 및 데이터 문서 읽기
- **쿼리 최적화**: 인덱스, 파티셔닝 및 성능 개선 사항 제안
- **설명 기능**: 학습 및 문서화를 위해 쿼리 구조 분석
- **테스트**: 테스트 쿼리 및 샘플 데이터 스크립트 생성 가능
- **스크립트 실행**: 데이터베이스에 즉시 사용 가능한 SQL 스크립트 작성

## 최상의 결과를 위한 팁

1. **맥락 제공**: 데이터베이스 스키마나 구조를 공유하십시오.
2. **구체성**: 필요한 데이터와 필터 조건을 명확하게 기재하십시오.
3. **데이터베이스 명시**: 사용 중인 SQL 방언을 지정하십시오.
4. **제약 사항 포함**: 데이터 볼륨, 기간 및 성는 요구 사항을 언급하십시오.
5. **형식 요청**: 특정 형식이 필요한 경우 쿼리 결과 형식을 요청하십시오.

## 출력 형식

다음을 받게 됩니다:

- **SQL 쿼리**: 주석이 포함된 프로덕션급 SQL 코드
- **설명**: 쿼리의 역할 및 작동 방식
- **성능 노트**: 최적화 팁 및 고려 사항
- **테스트 스크립트** (요청 시): 샘플 데이터 및 검증 쿼리
