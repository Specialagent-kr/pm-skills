# 가격 책정 전략 (Pricing Strategy)

가치 전달, 경쟁 포지셔닝 및 지불 의향에 근거한 가격 책정 전략을 설계합니다.

### 맥락

당신은 **$ARGUMENTS**를 위한 가격 책정 전략을 개발하고 있습니다.

사용자가 파일(경쟁사 가격, 설문 조사 데이터, 재무 모델 또는 사용량 데이터 등)을 제공하면 먼저 읽으십시오. 필요한 경우 웹 검색을 통해 경쟁사 가격을 조사하십시오.

### 지침

1. **전달되는 가치 이해**:
   - 핵심 가치 제안은 무엇인가?
   - 고객의 대안은 무엇이며 그 비용은 얼마인가?
   - 제품이 제공하는 정량적 성과는 무엇인가? (시간 절약, 매출 증대, 비용 절감 등)
   - 해당 가치를 기반으로 한 고객의 지불 의향(WTP)은 어느 정도인가?

2. **가격 책정 모델 평가** — 최적의 모델 추천:

   | 모델 | 적합한 경우 | 예시 |
   |---|---|---|
   | **정액제 (Flat-rate)** | 단순한 제품, 비용 예측이 쉬운 경우 | Basecamp ($99/월 정액) |
   | **사용자당 과금 (Per-seat)** | 협업 도구, 팀 제품 | Slack, Figma |
   | **사용량 기반 (Usage-based)** | 인프라, API 제품 | AWS, Twilio |
   | **계층형 (Tiered)** | 사용자 세그먼트가 뚜렷한 제품 | 대부분의 SaaS (무료/Pro/Enterprise) |
   | **프리미엄 (Freemium)** | 바이럴/네트워크 효과가 있는 제품 | Spotify, Notion |
   | **프리미엄 + 사용량** | 플랫폼 제품 | Vercel, OpenAI API |
   | **가치 기반 (Value-based)** | 영향력이 큰 엔터프라이즈 도구 | Salesforce, Palantir |

3. **경쟁사 가격 분석**:
   - 경쟁사의 가격 계층과 포함된 기능을 매핑합니다.
   - 당신의 제품이 어디에 위치하는지 식별합니다 (프리미엄, 미드마켓, 저가형).
   - 가격 공백이나 기회를 찾습니다.
   - 업계의 관행적인 가격 책정 방식에 유의하십시오.

4. **가격 구조 설계**:
   - **계층(Tiers)**: 차별화가 명확한 2-4개의 계층을 정의합니다.
   - **기능 제한(Feature gating)**: 어떤 기능을 어떤 계층에 넣을 것인가? (임의의 한도가 아닌 가치 지표를 사용하십시오)
   - **가치 지표(Value metric)**: 어떤 단위로 과금할 것인가? (사용자, 이벤트, 저장 용량, API 호출 등)
   - **앵커 가격(Anchor pricing)**: 가장 인기 있는 계층이 당연한 선택처럼 느껴지도록 설정합니다.
   - **연간 할인**: 일반적으로 월간 가격 대비 15-20% 할인율을 적용합니다.

5. **가격 민감도 추정**:
   - Van Westendorp 가격 민감도 측정 (설문 데이터가 있는 경우):
     - 너무 저렴함 → 품질 의심
     - 저렴함 → 좋은 가치
     - 비쌈 → 망설여지기 시작함
     - 너무 비쌈 → 구매하지 않음
   - 데이터가 없는 경우 경쟁사 가격과 전달되는 가치를 바탕으로 추정합니다.

6. **가격 실험 계획**:
   - 가격 페이지 A/B 테스트 (다른 가격대, 계층 이름, 기능 번들링 등)
   - 파운더가 직접 진행하는 영업 대화를 통해 지불 의향 테스트
   - 다른 가격 앵커를 사용한 랜딩 페이지 테스트
   - 가격대별 전환율에 대한 코호트 분석

7. **가격 책정 권장안 출력**:

   ```
   권장 모델: [모델 유형]
   가치 지표: [과금 단위]

   | 계층 | 가격 | 타겟 세그먼트 | 핵심 기능 | 포지셔닝 |
   |---|---|---|---|---|

   주요 가정:
   - [가정] → [테스트 방법]

   리스크:
   - [리스크] → [완화 방안]
   ```

단계별로 생각하십시오. 마크다운으로 저장하십시오. 출시 전 검증이 필요한 가정에는 표시를 하십시오.
