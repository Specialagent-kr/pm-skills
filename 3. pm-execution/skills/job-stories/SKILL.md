# 잡 스토리 (Job Stories)

'상황, 동기, 결과(When [situation], I want to [motivation], so I can [outcome])' 포맷을 사용하여 잡 스토리를 작성합니다. 사용자 상황과 성과(Outcome)에 초점을 맞춘 상세한 수용 기준을 포함하여 생성합니다.

**언제 사용하나:** 잡 스토리 작성, 사용자 상황과 동기 표현, JTBD(Jobs-to-be-Done) 스타일의 백로그 아이템 생성, 또는 역할(Role)보다는 사용자 맥락(Context)에 집중하고 싶을 때.

**인수:**

- `$PRODUCT`: 제품 또는 시스템 이름
- `$FEATURE`: 잡 스토리로 분해할 새로운 기능
- `$DESIGN`: 디자인 파일 링크 (Figma, Miro 등)
- `$CONTEXT`: 사용자 상황 또는 작업 시나리오

## 단계별 프로세스

1. **필요를 유발하는 사용자 상황 식별**
2. **사용자 행동의 근저에 있는 동기 정의**
3. **사용자가 달성하고자 하는 성과(Outcome) 명확화**
4. **JTBD 프레임워크 적용:** 역할이 아닌 '과업(Job)'에 집중
5. **성과 달성 여부를 검증하는 수용 기준(AC) 생성**
6. **관찰 가능하고 측정 가능한 언어 사용**
7. **디자인 목업 또는 프로토타입 연결**
8. **상세 수용 기준을 포함한 잡 스토리 출력**

## 스토리 템플릿

**제목:** [과업의 성과 또는 결과]

**내용:** [상황]일 때, 나는 [동기]하고 싶다. 그래야 [성과]할 수 있기 때문이다.

**디자인:** [디자인 파일 링크]

**수용 기준(Acceptance Criteria):**

1. [상황이 적절하게 인식됨]
2. [시스템이 원하는 동기를 지원함]
3. [진행 상황 또는 피드백이 시각화됨]
4. [성과가 효율적으로 달성됨]
5. [엣지 케이스가 원활하게 처리됨]
6. [통합 및 알림 기능이 작동함]

## 잡 스토리 예시

**제목:** 주간 간식 지출 추적

**내용:** 주간 간식비 예산을 준비할 때(상황), 지금까지 얼마나 썼는지 빨리 확인하고 싶다(동기). 그래야 주말이 되기 전에 돈이 떨어지지 않게 할 수 있기 때문이다(성과).

**디자인:** [Figma 링크]

**수용 기준:**

1. '주간 지출 개요' 섹션이 포함된 지출 요약 표시
2. 비용 기록 시 실시간 업데이트 반영
3. 진행 표시기 (주간 예산의 0-100%를 보여주는 진행 바)
4. 남은 예산을 눈에 띄는 색상으로 강조
5. 카테고리별로 분류된 상세 지출 로그 제공
6. 예산의 80% 도달 시 알림 발송
7. 목요일 저녁에 예산 90% 도달 시 주말 대비 알림 발송
8. 상세 내역에 쉽게 접근하고 탐색 가능

## 출력물

- 해당 기능에 대한 전체 잡 스토리 세트
- '상황... 동기... 결과...' 포맷을 준수하는 스토리
- 성과에 집중한 6-8개의 수용 기준
- 사용자 상황과 동기를 강조한 내용
- 디자인 및 프로토타입에 대한 명확한 링크
