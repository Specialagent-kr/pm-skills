# 테스트 시나리오

사용자 스토리로부터 테스트 목표, 시작 조건, 사용자 역할, 단계별 테스트 행동 및 기대 결과를 포함하는 종합적인 테스트 시나리오를 생성합니다.

**언제 사용하나:** QA 테스트 케이스 작성, 테스트 계획 수립, 수용 테스트 시나리오 정의 또는 사용자 스토리 구현 검증 시.

**인수:**

- `$PRODUCT`: 제품 또는 시스템 이름
- `$USER_STORY`: 테스트할 사용자 스토리 (제목 및 수용 기준)
- `$CONTEXT`: 추가적인 테스트 맥락 또는 제약 사항

## 단계별 프로세스

1. **사용자 스토리 및 수용 기준 검토**
2. **테스트 목표 정의** - 검증할 구체적인 행동 파악
3. **시작 조건 설정** - 시스템 상태, 데이터 설정, 구성 등
4. **사용자 역할 식별** - 누가 테스트 행동을 수행하는가
5. **테스트 단계 생성** - 상호작용을 단계별로 분해
6. **기대 결과 정의** - 각 단계 후 관찰 가능한 결과
7. **엣지 케이스 고려** - 유효하지 않은 입력, 경계 조건 등
8. **상세 테스트 시나리오 출력** - QA 실행 가능 수준

## 테스트 시나리오 예시

**테스트 시나리오:** 제품 페이지에서 최근에 본 상품 섹션 확인

**테스트 목표:** '최근에 본 상품' 섹션이 올바르게 표시되고 현재 상품은 제외되는지 확인합니다.

**시작 조건:**

- 사용자가 로그인했거나 브라우저 기록이 활성화됨
- 사용자가 현재 세션에서 적어도 2개의 상품을 보았음
- 사용자가 이전에 본 항목과 다른 제품 페이지에 있음

**사용자 역할:** 온라인 쇼핑객

**테스트 단계:**

1. 임의의 제품 페이지로 이동 → 하단에 이전에 본 항목과 함께 섹션이 나타나야 함
2. 페이지 하단으로 스크롤 → '최근에 본 상품' 섹션이 상품 카드와 함께 표시됨
3. 상품 썸네일 확인 → 이미지, 제목, 가격이 올바르게 표시됨
4. 현재 상품 확인 → 현재 보고 있는 상품은 최근에 본 목록에 포함되지 않음
5. 상품 카드 클릭 → 해당하는 상품 페이지로 이동함

**기대 결과:**

- 이전 상품을 적어도 하나 본 경우에만 최근에 본 섹션이 나타남
- 섹션에 4-8개의 완성된 정보가 담긴 상품 카드가 표시됨
- 현재 상품은 목록에서 제외됨
- 각 카드에 "X분/시간 전 확인" 타임스탬프 표시
- 카드 클릭 시 정확한 상품 페이지로 이동
- 성능: 섹션이 2초 이내에 로드됨

## 출력물

- 각 수용 기준에 대한 종합적인 테스트 시나리오
- 사용자 스토리의 의도와 일치하는 명확한 테스트 목표
- 상세한 단계별 테스트 행동
- 각 단계 후 관찰 가능한 기대 결과
- 엣지 케이스 및 에러 시나리오 포함
- QA 팀의 실행 및 문서화에 바로 활용 가능
