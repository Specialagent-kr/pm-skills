# 사용자 스토리 (User Stories)

3 C(Card, Conversation, Confirmation) 및 INVEST 기준을 따르는 사용자 스토리를 생성합니다. 상세 설명, 디자인 링크 및 수용 기준을 포함합니다.

**언제 사용하나:** 사용자 스토리 작성, 기능을 스토리로 분해, 백로그 아이템 생성 또는 수용 기준 정의 시.

**인수:**

- `$PRODUCT`: 제품 또는 시스템 이름
- `$FEATURE`: 스토리로 분해할 새로운 기능
- `$DESIGN`: 디자인 파일 링크 (Figma, Miro 등)
- `$ASSUMPTIONS`: 주요 가설 또는 맥락

## 단계별 프로세스

1. **제공된 디자인과 맥락을 바탕으로 기능 분석**
2. **사용자 역할 및 별도의 사용자 여정 식별**
3. **3 C 프레임워크 적용:**
   - Card: 단순한 제목과 한 문장 요약
   - Conversation: 의도에 대한 상세한 논의
   - Confirmation: 명확한 수용 기준(AC)
4. **INVEST 기준 준수:** Independent(독립적), Negotiable(협상 가능), Valuable(가치 있는), Estimable(추정 가능), Small(작은), Testable(테스트 가능)
5. **초등 교육 수준에서 이해할 수 있는 쉬운 언어 사용**
6. **시각적 참조를 위해 디자인 파일 연결**
7. **구조화된 형식으로 사용자 스토리 출력**

## 스토리 템플릿

**제목:** [기능 이름]

**내용:** [사용자 역할]로서, 나는 [행동]하고 싶다. 그래야 [이득]할 수 있기 때문이다.

**디자인:** [디자인 파일 링크]

**수용 기준(Acceptance Criteria):**

1. [명확하고 테스트 가능한 기준]
2. [관찰 가능한 행동]
3. [시스템이 올바르게 검증함]
4. [엣지 케이스 처리]
5. [성능 또는 접근성 고려 사항]
6. [통합 지점]

## 사용자 스토리 예시

**제목:** 최근에 본 상품 섹션

**내용:** 온라인 쇼핑객으로서, 나는 제품 페이지에서 '최근에 본 상품' 섹션을 보고 싶다. 그래야 내가 고려했던 상품들을 쉽게 다시 확인할 수 있기 때문이다.

**디자인:** [Figma 링크]

**수용 기준:**

1. 이전에 적어도 1개의 상품을 본 모든 사용자에게 제품 페이지 하단에 '최근에 본 상품' 섹션이 표시됨
2. 세션의 첫 번째 상품 페이지를 방문 중인 사용자에게는 표시되지 않음
3. 현재 보고 있는 상품은 목록에서 제외됨
4. 해당 섹션에 이미지, 제목, 가격이 포함된 상품 카드 또는 썸네일이 표시됨
5. 각 상품 카드에 언제 확인했는지 표시함 (예: '5분 전 확인')
6. 상품 카드를 클릭하면 해당하는 상품 페이지로 이동함

## 출력물

- 해당 기능에 대한 전체 사용자 스토리 세트
- 각 스토리에 제목, 내용, 디자인 링크 및 4-6개의 수용 기준 포함
- 각 스토리는 독립적이며 어떤 순서로든 개발 가능함
- 한 번의 스프린트 사이클 내에 완료 가능한 크기
- 관련 디자인 문서 참조 포함
