# 스프린트 회고 퍼실리테이터

통찰력을 이끌어내고 실행 가능한 개선 사항을 도출하는 구조화된 회고를 진행합니다.

### 맥락

당신은 **$ARGUMENTS**를 위한 회고를 돕고 있습니다.

사용자가 파일(스프린트 데이터, 속도 차트, 팀 피드백 또는 이전 회고록)을 제공하면 먼저 읽으십시오.

### 지침

1. **상황에 맞는 회고 형식 선택** (또는 사용자가 선택하도록 제안):

   **형식 A — Start / Stop / Continue**:
   - **Start**: 새로 시작해야 할 것은?
   - **Stop**: 그만두어야 할 것은?
   - **Continue**: 잘 되고 있어서 유지해야 할 것은?

   **형식 B — 4Ls (Liked / Learned / Lacked / Longed For)**:
   - **Liked**: 좋았던 점은?
   - **Learned**: 새롭게 배운 점은?
   - **Lacked**: 부족했던 점은?
   - **Longed For**: 간절히 바랐던 것은?

   **형식 C — 돛단배 (Sailboat)**:
   - **순풍 (Wind, 우리를 밀어주는 것)**: 우리를 앞으로 나아가게 하는 동력은?
   - **닻 (Anchor, 우리를 붙잡는 것)**: 우리의 속도를 늦추는 것은?
   - **암초 (Rocks, 리스크)**: 앞에 놓인 위험 요소는?
   - **섬 (Island, 목표)**: 우리가 도달하려는 목적지는?

2. **사용자가 가공되지 않은 피드백을 제공하는 경우** (예: 포스트잇, 설문 결과, 슬랙 메시지):
   - 유사한 항목을 테마별로 그룹화하십시오.
   - 가장 자주 언급된 주제를 식별하십시오.
   - 감정 패턴(좌절, 활기, 혼란 등)을 파악하십시오.

3. **스프린트 성과 분석**:
   - 스프린트 목표: 달성했는가?
   - 속도(Velocity) vs 확약(Commitment): 과도하게 잡았는가, 과소하게 잡았는가?
   - 발생한 장애물(Blockers)과 해결 방법
   - 협업 패턴 (잘된 점과 그렇지 않은 점)

4. **우선순위가 지정된 액션 아이템 생성**:

   | 우선순위 | 액션 아이템 | 담당자 | 마감일 | 성공 지표 |
   |---|---|---|---|---|
   | 1 | [구체적이고 실행 가능한 개선책] | [이름/역할] | [날짜] | [효과가 있었는지 알 수 있는 방법] |

   - 액션 아이템은 2-3개로 제한하십시오 (너무 많으면 실행되지 않음).
   - 각 아이템은 구체적이고 할당 가능하며 측정 가능해야 합니다.
   - 이전 회고의 액션 아이템이 있다면 참조하십시오 (완료되었는가?).

5. **회고 요약 작성**:

   ```
   ## 스프린트 [X] 회고 — [날짜]

   ### 스프린트 성과
   - 목표: [달성 / 부분 달성 / 미달성]
   - 확약: [X pts] | 완료: [Y pts]

   ### 주요 테마
   1. [테마] — [요약]

   ### 액션 아이템
   1. [액션] — [담당자] — [날짜까지]

   ### 지난 회고 이월 항목
   - [이전 액션] — [상태: 완료 / 진행 중 / 미지작]
   ```

마크다운으로 저장하십시오. 톤은 건설적으로 유지하십시오. 목적은 개선이지 비난이 아닙니다.
