# 제품 실행 (pm-execution)

실행 및 제품 관리 스킬: PRD 작성, OKR 수립, 로드맵 관리, 스프린트 계획, 사전 부검(Pre-mortem), 이해관계자 지도, 사용자 스토리, 우선순위 프레임워크 등.

## 스킬 (15)

- **brainstorm-okrs** — 회사의 목표와 정렬된 팀 단위의 OKR을 수립합니다.
- **create-prd** — 요약, 배경, 목표, 시장 세그먼트, 가치 제안, 솔루션 상세 내용 및 출시 계획을 포함하는 8개 섹션의 종합 PRD를 작성합니다.
- **dummy-dataset** — 테스트를 위한 현실적인 더미 데이터셋을 생성합니다(CSV, JSON, SQL, Python 스크립트 형식 지원).
- **job-stories** — '상황, 동기, 결과' 포맷과 상세한 수용 기준(Acceptance Criteria)을 포함하는 잡 스토리를 작성합니다.
- **outcome-roadmap** — 기능 중심의 로드맵을 성과(Outcome) 중심의 로드맵으로 변환합니다.
- **pre-mortem** — PRD에 대한 사전 부검 분석을 수행하여 리스크를 파악합니다.
- **prioritization-frameworks** — 공식, 사용 시점 가이드, 템플릿을 포함한 9가지 우선순위 지정 프레임워크 참고 가이드입니다.
- **release-notes** — 티켓, PRD 또는 변경 로그를 바탕으로 사용자용 출시 노트를 생성합니다.
- **retro** — 구조화된 스프린트 회고를 진행합니다.
- **sprint-plan** — 가용성 추정, 스토리 선택, 의존성 매핑 및 리스크 식별을 포함한 스프린트 계획을 수립합니다.
- **stakeholder-map** — 영향력/관심도 그리드를 사용하여 이해관계자 지도를 구축하고 커뮤니케이션 전략을 수립합니다.
- **summarize-meeting** — 회의 녹취록을 날짜, 참석자, 주제, 요약 및 액션 아이템이 포함된 구조화된 템플릿으로 요약합니다.
- **test-scenarios** — 사용자 스토리에서 테스트 목표, 시작 조건, 역할, 단계별 행동 및 기대 결과를 포함하는 종합적인 테스트 시나리오를 생성합니다.
- **user-stories** — 3 C(Card, Conversation, Confirmation) 및 INVEST 기준을 따르는 사용자 스토리를 작성합니다.
- **wwas** — Why-What-Acceptance 포맷의 백로그 아이템을 작성합니다.
